1. Description of the Use Case
Target users are computer science students preparing for internships and placements, and administrators who
manage assessments. Students today juggle multiple fragmented platforms — coding sites, MCQ banks and aptitude
portals — with no unified tracking or placement-specific simulation. Our platform centralises coding problems, SQL
exercises, subject MCQs (DSA, CN, OS, OOP, DBMS) and aptitude questions into a single environment with
customisable timed tests, Docker-backed code execution, and analytics dashboards that help students identify weak
areas and improve placement readiness. Administrators manage question banks and configure assessments through
a dedicated console.
2. Key Functionalities
Core Features
 Customisable timed test builder — select topics, difficulty, and question types
 Coding evaluation with test cases in an isolated Docker sandbox
 Subject MCQs (DSA, OS, CN, OOP, DBMS) with instant scoring and explanations

System Architecture
The system is decomposed into five logical layers. Every service runs as an independent Docker container.
Asynchronous communication via a message broker decouples the submission evaluator from the code execution
sandbox, ensuring fault isolation and horizontal scalability.
